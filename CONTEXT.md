# X-List Market Summary

A program that summarizes tweets from a fixed X (Twitter) list, grouped by fixed-time windows aligned to Japanese stock-market session labels, and sends each summary to Telegram.

## Language

**X List**:
A single, fixed X list whose tweets are the source of all summaries.
_Avoid_: List (ambiguous), Timeline, Feed

**Summary Window**:
A fixed clock-time interval over which tweets are collected and summarized. Named after the market session it roughly aligns to, but driven by clock time, not the trading calendar.
_Avoid_: Session (reserved for actual trading sessions), Time slot, Period

**Summary Window** set (fixed, JST):
- 朝場: 06:00-12:30 → sent at 12:30
- 昼場: 12:30-16:30 → sent at 16:30
- 夜場: 16:30-24:00 → sent at 24:00 (together with Daily)
- Daily: 00:00-24:00 → sent at 24:00

Window boundaries are **left-closed, right-open** `[start, end)`: a post timestamped exactly at a boundary belongs to the later window. No 06:00 send exists.

**Market Session**:
The actual trading hours of Japanese exchanges (前場 9:00-11:30, 後場 12:30-15:00, PTS 8:30-16:00). Used only as a naming reference for Summary Windows, never as the trigger.
_Avoid_: 立会, Session (when meaning the window)

**Window Summary**:
The structured output sent to Telegram for one Summary Window. Always has four sections: 主要ニュース, 銘柄・テーマ動向, センチメント, 注目ポイント. No links to source tweets.
_Avoid_: Report, Digest, Recap

**Daily Summary**:
A summary covering the full 24-hour window (00:00-24:00), sent at 24:00. Generated via the hybrid method: the day's raw posts from the Tweet Store plus the three intraday Window Summaries are fed together to the Summarizer. Follows the same Summary Schema, scaled to a full-day view.
_Avoid_: Full-day summary, End-of-day recap

**Summary Schema**:
The fixed four-section structure every Window Summary must follow. Tickers and numeric values are preserved verbatim from source tweets; sentiment is quantified as 強気/中立/弱気 with good/bad item counts.
_Avoid_: Template, Format

**Summarizer**:
The LLM that condenses a window's tweets into a Telegram-bound summary. Currently Gemini 3.5 Flash.
_Avoid_: Model, Generator

**Fetch Cursor (since_id)**:
The ID of the most recently fetched post. The next fetch requests only posts newer than this ID, so already-billed posts are not re-fetched. Persisted across runs.
_Avoid_: Watermark, Offset, Last-seen ID

**Tweet Store**:
The local persistence holding fetched posts and per-window summaries. Implemented as date-scoped JSON files (`store/YYYY-MM-DD.json`) plus a single `cursor.json` holding the Fetch Cursor. Window slicing is an in-memory filter over the current day's file. Old files are archived/deleted on rotation.
_Avoid_: Cache, Database (too implementation-specific)

**Error Notification**:
A Telegram message sent to the same chat as summaries, only when a send-time job fails completely (retry-exhausted) or when polling fails 3 times consecutively. Transient failures are absorbed by retries and logged silently.
_Avoid_: Alert, Alarm

## Decisions

- Fetching is done via SocialData.tools' `Get List Tweets` endpoint on a **public** X List. See [ADR-0001](docs/adr/0001-socialdata-for-list-tweets.md).
- Summaries are generated by **Gemini 3.5 Flash**. See [ADR-0002](docs/adr/0002-gemini-3-5-flash-for-summarization.md).
- The program runs on a **VPS via cron**, polling the list every **15 minutes** and accumulating posts into a local Tweet Store. See [ADR-0003](docs/adr/0003-vps-cron-polling-architecture.md).
- Telegram delivery: single chat, auto-split on the 4096-char limit, section-structured output with no source links.
- Daily Summary uses the hybrid method (raw posts + intraday Window Summaries as input).
- Failure handling: 3-retry exponential backoff; user-facing Error Notification only on complete window-send failure or 3 consecutive poll failures; `cursor.json` written atomically.
- Post filter: replies to other accounts are excluded from summarization; retweets and quote-tweets are included.
- Cron entries: `*/15 * * * *` (poll), `30 12 * * *` (朝場 send), `30 16 * * *` (昼場 send), `0 0 * * *` (夜場 + Daily send). All times JST; the VPS timezone must be set to Asia/Tokyo.
